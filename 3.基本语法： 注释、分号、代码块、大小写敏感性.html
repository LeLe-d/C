阶段一：基础入门 - 基本语法

理解 C 语言的基础语法规则是编写正确、可读代码的第一步。这些规则就像语言的“语法”，规定了如何组织代码元素。以下是几个最核心的语法要点：

1. 注释 (Comments)

注释是代码中用于解释说明的部分，它们会被编译器完全忽略，不会影响程序的执行。注释的主要目的是提高代码的可读性，方便自己和他人理解代码的意图。

C 语言支持两种类型的注释：

  • 单行注释： 以 // 开始，直到该行结束的所有内容都是注释。 
    int score = 100; // 这是单行注释，用来解释 score 变量的初始值
  • 多行注释： 以 /* 开始，以 */ 结束。可以跨越多行，也可以在一行内使用。 
    /*
这是一个
多行注释的例子。
它可以用来写较长的说明。
*/
float price = 99.9; /* 也可以像这样在一行内使用 */

    注意： 在 C89 标准中只支持 /* ... */，// 是 C99 标准引入并被广泛支持的。

2. 分号 (Semicolon ;)

在 C 语言中，分号扮演着 语句结束符 的角色。它告诉编译器一个独立的语句到此结束。绝大多数 C 语句都必须以分号结尾。

忘记加分号是初学者常见的编译错误。

int age = 25;       // 声明并初始化变量，需要分号
printf("Hello\n"); // 函数调用语句，需要分号
age = age + 1;    // 赋值语句，需要分号
return 0;         // return 语句，需要分号

// 例外情况（不需要分号的常见场景）：
// 1. 预处理指令 (如 #include, #define)
// #include <stdio.h>  <-- 没有分号
// 2. 函数定义的头部
// int main() {        <-- 没有分号
// 3. 控制结构 (如 if, while, for) 的条件部分之后
// if (age > 18) {    <-- 没有分号
// 4. 代码块的结束花括号 } 之后
// }                 <-- 没有分号

3. 代码块 (Code Blocks { })

代码块是由一对花括号 { } 包围起来的一系列语句。代码块的作用是将多条语句组合在一起，形成一个逻辑单元。

代码块常用于以下场景：

  • 函数体： 函数的所有代码都必须放在一个代码块中。
  • 控制流语句： 如 if, else, while, for, do-while 等，当需要执行多条语句时，必须使用代码块将这些语句括起来。
  • 定义局部变量的作用域。
int main() { // 函数体的开始，是一个代码块

    int temperature = 15;

    if (temperature < 20) { // if 语句关联的代码块开始
        printf("有点冷。\n");
        printf("建议穿外套。\n"); // 这两条语句被组合在 if 的块内
    } // if 语句关联的代码块结束
    else { // else 语句关联的代码块开始
         printf("天气不错。\n");
    } // else 语句关联的代码块结束

    return 0;
} // 函数体的结束

注意： 代码块的结束花括号 } 后面不需要加分号。

4. 大小写敏感性 (Case Sensitivity)

C 语言是严格区分大小写的。这意味着同一个字母的大写形式和小写形式被视为不同的字符。

这适用于：

  • 标识符 (Identifiers)： 变量名、函数名、结构体名等。例如，myVariable, myvariable, 和 MyVariable 是三个完全不同的标识符。
  • 关键字 (Keywords)： C 语言的关键字 (如 int, float, if, else, while, return) 必须全部小写。写成 Int, IF, RETURN 等都是错误的。
int count = 0;
int Count = 10; // 'count' 和 'Count' 是两个不同的变量

printf("%d\n", count); // 正确调用 printf 函数
// Printf("%d\n", Count); // 错误！没有名为 Printf 的函数 (除非自己定义)

if (count != Count) { // 正确使用关键字 if
    // ...
}
// IF (count != Count) { // 错误！IF 不是关键字
    // ...
// }

建议： 养成统一的命名习惯（例如，变量名和函数名使用小写字母和下划线 lower_case_with_underscores，或者驼峰命名法 camelCase），并始终保持一致，以避免因大小写混淆导致的错误。

总结： 掌握注释、分号、代码块和大小写敏感性这些基本语法规则，是编写任何 C 程序的基础。务必在早期就养成严格遵守这些规则的习惯。
